## Pitfalls (specific to this section)

- **Don't leak system claims into §3.** Everything in this section must be about the *empirical* contribution — the corpus and the failure-mode analysis. The tool goes in §4–§7.
- **Every number needs a source.** `output/key-numbers.md` is not optional. Reviewers WILL grep for consistency.
- **Anonymized examples if the venue requires double-blind.** Check the FSE 2027 CfP when it publishes; strip any repo URLs from the bug examples in the draft accordingly.
